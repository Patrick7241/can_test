# 🎮 大众ID.4 CAN总线驾驶游戏

真实的游戏画面 + CAN总线通信 = 沉浸式驾驶体验！

## ✨ 游戏特性

### 🎯 游戏玩法
- **俯视图驾驶**: 经典的俯视角驾驶视角
- **三车道系统**: A/D键切换车道
- **障碍物躲避**: 躲避随机生成的障碍物
- **碰撞检测**: 撞到障碍物游戏结束
- **难度递增**: 随时间增加，障碍物速度加快

### 🚗 三种障碍物类型
1. **红色车辆** - 其他车辆，需要躲避
2. **橙色锥桶** - 道路维护标志
3. **灰色路障** - 带黄色条纹的路障

### 📡 CAN总线集成
- **真实信号**: 游戏操作实时生成CAN信号
- **实时显示**: HUD显示车速、档位、油门、刹车等
- **ECU模拟**: 后台运行车辆ECU，每100ms发送CAN帧
- **监听器**: 仪表盘监听器接收CAN信号

## 🚀 快速开始

### 安装依赖

游戏使用 [ebiten](https://ebiten.org/) 游戏引擎：

```bash
# 已自动安装，如果需要重新安装：
go get github.com/hajimehoshi/ebiten/v2
```

### 编译运行

```bash
cd vehicles/volkswagen_id4
go build -o game_demo signals.go canbus.go vehicle_ecu.go can_listeners.go game_demo.go
./game_demo
```

或者直接运行：

```bash
go run signals.go canbus.go vehicle_ecu.go can_listeners.go game_demo.go
```

## 🎮 操作说明

### 主菜单
- **[空格]** 开始游戏

### 游戏中
| 按键 | 功能 | CAN信号 |
|------|------|---------|
| **W** | 加速 | 油门增加 → 0x3EB |
| **S** | 刹车 | 刹车增加 → 0x3EB |
| **A** | 左换道 | 转向左 → 0x3DA |
| **D** | 右换道 | 转向右 → 0x3DA |
| **空格** | 松开油门/刹车 | 油门/刹车减少 |
| **P** | 暂停/继续 | - |

### 游戏结束
- **[R]** 重新开始
- **[Q]** 退出游戏

## 🎯 游戏技巧

### 基础技巧
1. **提前规划**: 看到障碍物提前换道
2. **保持速度**: 速度越快分数越高
3. **平稳操作**: 避免频繁急转
4. **专注力**: 随着时间推移难度增加

### 高级技巧
1. **节奏感**: 找到障碍物生成的节奏
2. **预判**: 提前判断下一个障碍物位置
3. **极限操作**: 在最后一刻换道获得更高分
4. **速度管理**: 适当减速更容易控制

## 🔧 技术细节

### 游戏架构

```
┌──────────────┐
│  Game Loop   │  60 FPS
└──────┬───────┘
       │
       ├─→ 用户输入 ──────────┐
       │                      ↓
       ├─→ 更新逻辑       VehicleECU
       │   - 玩家位置          ↓
       │   - 障碍物移动    生成CAN帧
       │   - 碰撞检测          ↓
       │                   CANBus
       │                      ↓
       ├─→ 渲染画面      DashboardListener
       │   - 道路              ↓
       │   - 车辆          更新HUD显示
       │   - 障碍物
       │   - HUD
       └─────────────────────┘
```

### 碰撞检测

使用AABB (Axis-Aligned Bounding Box) 碰撞检测：

```go
func (obj *GameObject) Intersects(other *GameObject) bool {
    x1, y1, x2, y2 := obj.Bounds()
    ox1, oy1, ox2, oy2 := other.Bounds()

    return x1 < ox2 && x2 > ox1 && y1 < oy2 && y2 > oy1
}
```

### CAN信号生成

每次操作都会触发CAN信号：

```go
// W键加速
g.vehicleECU.HandleInput('w')
  ↓
油门位置增加
  ↓
生成0x3EB CAN帧
  ↓
发送到CAN总线
  ↓
仪表盘监听器接收
  ↓
HUD显示更新
```

### 性能优化

- **60 FPS**: 流畅的游戏体验
- **对象池**: 障碍物复用避免频繁GC
- **异步CAN**: CAN总线在后台线程运行
- **批量渲染**: 一次性渲染所有对象

## ❓ 常见问题

**Q: 游戏窗口太小/太大？**
A: 窗口支持调整大小，拖动窗口边缘即可。

**Q: 游戏卡顿？**
A: 确保没有其他占用GPU的程序运行。游戏运行在60FPS。

**Q: 如何看到CAN帧详情？**
A: 游戏集成了CAN总线，按P暂停后可以在终端看到CAN统计。

**Q: 障碍物太快了！**
A: 难度随时间增加，初期速度较慢，适合练习。

**Q: 可以修改游戏参数吗？**
A: 可以！编辑 `game_demo.go`：
- `spawnRate`: 障碍物生成频率
- `VelY`: 障碍物移动速度
- `roadWidth`: 道路宽度
- 等等
